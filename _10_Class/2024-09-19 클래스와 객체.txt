클래스는 레시피는 만드는 것

<level 1>
▶쭈꾸미 레시피는 실제 사람이 먹을 음식이라고 할 수있을까??
▶없다
▶이유: 레시피는 그 음식을 만들 떄 필요한 과정일 뿐 사람이 먹을 순 없다.
	
▶레시피는 왜 만들었을까??
▶이유: 레시피는 그 음식을 어떻게 만들지? 어떻게 먹기 위해 만들어진 것
▶왜? 만들었을까? 재사용성을 높이기 위해서
 
▶레시피와 실제 만들어진 음식은 어떤 차이가 있을까??
▶레시피는 정의서 설명서 실제 만들어진 음식을 사용하고자 하는 대상이 필요, 
▶이유: 레시피는 그 음식을 만들기 위해 어떻게 만들지 가이드란인을 정해놓은 것

<level 2>
▶레시피는 1개만 있고 레시피를 통해 만들어진 음식도 1개라고 말할 수 있을까?
▶있다. 레시피를 재사용해서 1개의 음식이 아닌 여러가지 음식을 만들 수 있다.
▶없다. 하나의 레시피로 여러가지 음식을 만들 수 없다.

▶레시피가 있다면 음식도 만들진 것일까? 
??
▶서로 다른 쭈구미 레시피로 각각 요리를 했다면 음식은 쭈꾸미라는 성격을 갖는다. 
▶이 음식은 서로 같은 음식이라고 할 수 있을까?
▶레시피가 다르면 다른 음식이라고 볼 수 있다.

<level 3>
▶레시피는 여러 사람과 공유할 수 있다, 이떄 레시피는 하나이다.
   레시피를 참조하는 사람도 1명일까?
▶레시피를 여러사람과 공유할 수 있으므로 참조하는 사람은 1명이 아닌 여러명이다.
공유할 떄 좋은 점은 자원의 낭비를 줄일 수 있다.또는 효율적(큰 장점)
프로그램에서 메모리 적게 사용하는 것이다. 
▶여러 사람이 하나의 레시피를 참조할 떄
   어떤 사람이 레시피를 수정한다면 다른 사람은 수정된 레시피를 볼까? 
   아니면 수정되지 않은 상태의 레시피를 볼까?
▶공유는 자원을 절약하기 위해 
   공유한다면 어떤 사람이 내용을 바꾸면 다른 사람도 변경된 내용을 보게된다
   메모리 사용용량 줄임
   CPU 사용량 줄임
클래스를 만들 떄 독립적으로 사용할지 공유할지 정해야된다.


<level 4>

클래스는 new연산자로 객체를 만든다.
	class  →  객체
ex) new Random → Random 
ex) new Scanner → Scanner
클래스를 가지고 객체를 만든다.
